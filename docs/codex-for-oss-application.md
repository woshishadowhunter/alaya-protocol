# Codex for Open Source Application Readiness

## Do not submit immediately

The program asks for active maintenance, usage, adoption, or clear ecosystem importance. A newly created repository should first establish truthful public evidence.

## Evidence target

- Three or more tagged releases over at least six weeks.
- Ten independent users or three external projects running the SDK.
- Public issues showing real use, including resolved issues.
- At least two external contributions or framework adapters.
- A reproducible evaluation and documented maintenance activity.

## Repository importance draft (adapt after evidence exists)

Alaya provides a framework-neutral, auditable experience lifecycle for long-running AI agents. Unlike transcript or preference memory, it records contextual lessons with independent evidence, contradictions, applicability boundaries, decay, and explainable activation. The project includes a portable schema, dependency-light Python SDK, local storage, Skill, and deterministic evaluations. Add verified stars, users, downstream projects, releases, issues, and contributors here before submitting.

## API credit use draft

Credits would support maintainership workflows: issue triage, PR review, release preparation, security-oriented review, compatibility testing, and opt-in evaluation of candidate lesson extraction across realistic agent traces. Deterministic policy and tests remain model-independent; API usage would evaluate extraction quality and help maintain adapters and documentation.

## Other information draft

Alaya is inspired by Yogacara seed and activation concepts but makes no claim of machine consciousness. Its contribution is engineering: experiences remain inspectable and revisable; contradictory evidence is preserved; one reflection cannot silently change behavior; and local storage, export, and deletion are first-class capabilities.
