# Alaya Protocol v0.1 → v0.2 Upgrade Design

## Scope

Five high-impact upgrades to the Alaya Protocol, all backward-compatible.

## 1. Pluggable Strategy System

Replace hardcoded decision logic with protocol-based injection points.

- `PolicyProtocol`: interface with `should_promote()`, `should_deprecate()`, `confidence_delta()`, `decay_factor()`
- `EvolutionPolicy`: existing dataclass now implements `PolicyProtocol`
- `ExperienceEngine.__init__` accepts `policy: PolicyProtocol | None`

## 2. Semantic Retrieval

Optional embedding-based retrieval alongside default lexical matching.

- `RetrievalBackend` protocol: `encode(texts)` + `similarity(a, b)`
- `LexicalRetrieval`: zero-dependency default, identical to v0.1 behavior
- `activate()` gains optional `backend: RetrievalBackend | None` parameter
- No new dependencies; semantic backends injected by caller

## 3. Counterexample Penalty

Wire `counterexamples` into activation scoring.

- When context tokens overlap with any counterexample, relevance is multiplied by `counterexample_penalty` (default 0.3)
- Configurable via `EvolutionPolicy.counterexample_penalty: float`

## 4. Storage Scalability

Add filtered, paginated query methods.

- `list_active(since_days)`: only active + recent seeds
- `count()`: total without materializing all
- `iter_all(batch_size)`: cursor-based batch iteration
- `export_json()` uses `iter_all()` internally

## 5. Activation Structured Fields

Split `explanation` string into typed fields.

- `Activation` gains: `confidence: float`, `recency: float`, `matched_terms: tuple[str, ...]`
- `explanation` preserved for human readability
- Factory method `Activation.create()` builds both

## Backward Compatibility

- All existing public APIs unchanged
- `EvolutionPolicy` constructor unchanged
- `activate()` without `backend` uses lexical matching (v0.1 behavior)
- `store.list()` unchanged
- `Activation` fields are additive; `explanation` still present

## Files Changed

| File | Changes |
|------|---------|
| `engine.py` | +PolicyProtocol, +RetrievalBackend, +LexicalRetrieval, counterexample penalty, Activation restructure |
| `models.py` | EvolutionPolicy implements PolicyProtocol methods |
| `store.py` | +list_active, +count, +iter_all |
| `cli.py` | activate uses list_active, export uses iter_all |
| `__init__.py` | Export new public symbols |
| `tests/` | Full coverage of all new logic |
