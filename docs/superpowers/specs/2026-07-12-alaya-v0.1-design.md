# Alaya Protocol v0.1 Design

## Outcome

Alaya is a framework-neutral experience-evolution layer for long-running AI agents. It lets an agent turn repeated work and life interactions into auditable experience seeds that can later influence decisions. It does not claim consciousness and does not treat a single model reflection as truth.

## First user and use case

The first user is an agent application developer. The first reference experience is recurring work decisions and project delivery: context, chosen action, observed outcome, user feedback, extracted lesson, applicability boundaries, and counterevidence.

## Architecture

The repository contains:

- A versioned JSON Schema defining portable experience seeds.
- A dependency-light Python SDK with domain models, deterministic evidence scoring, activation, conflict handling, decay, and SQLite persistence.
- An optional reflector interface. Applications may use an LLM to propose lessons; deterministic policy controls whether proposals become active seeds.
- A Codex-compatible `learn-from-experience` skill that invokes the CLI scripts without embedding the protocol logic in prompts.
- Examples and a small behavior-change evaluation.

## Experience lifecycle

1. `observe`: record a context, action, outcome, and optional feedback.
2. `plant`: create a candidate seed with explicit applicability and provenance.
3. `reinforce`: add supporting or contradicting evidence.
4. `activate`: rank applicable seeds for a new context using lexical relevance, confidence, recency, and status.
5. `evolve`: promote candidates after sufficient evidence, weaken contradicted seeds, and deprecate unreliable seeds.

Default policy requires two independent supporting observations before a candidate becomes active. A contradiction reduces confidence but remains auditable. Seeds never silently overwrite one another.

## Data model

Each seed has an ID, lesson, context tags, action guidance, applicability, counterexamples, confidence, support and contradiction counts, status, provenance events, created/updated timestamps, and last activation timestamp. Raw private transcripts are not required; callers decide what summarized evidence to store.

## Safety and privacy

- Local SQLite and JSON are the defaults; no network calls are required.
- Every seed is inspectable, exportable, and deletable.
- LLM output is treated as an untrusted proposal.
- The SDK validates bounded scores, allowed statuses, and evidence polarity.
- The README prohibits covert profiling and high-stakes autonomous use without human review.

## Testing and evaluation

Unit tests cover schema round trips, promotion thresholds, contradictions, decay, activation relevance, persistence, export/delete, and CLI behavior. A deterministic project-decision demo compares a baseline agent context with an Alaya-enriched context and reports whether the correct prior lesson is activated while irrelevant lessons remain inactive.

## Success criteria

- Installable with Python 3.10+ and no runtime dependency beyond the standard library.
- A developer can plant, reinforce, activate, inspect, export, and delete seeds in under ten lines per operation.
- All tests and skill validation pass locally.
- Documentation clearly separates Yogacara inspiration from empirical engineering claims.
