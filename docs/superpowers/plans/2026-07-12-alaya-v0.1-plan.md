# Alaya Protocol v0.1 Implementation Plan

## Task 1: Package and schema

Create `pyproject.toml`, package exports, and `protocol/experience-seed.schema.json`. Add a schema/model round-trip test first, verify failure, then implement immutable dataclasses with JSON serialization and validation.

## Task 2: Evolution engine

Add failing tests for candidate promotion after two supports, confidence reduction after contradiction, deterministic decay, and preservation of provenance. Implement `EvolutionPolicy` and `ExperienceEngine` minimally until green.

## Task 3: Activation

Add failing tests proving relevant active seeds rank above unrelated or candidate seeds. Implement token-based relevance combined with confidence and recency; return a score explanation with every activation.

## Task 4: SQLite store and CLI

Add failing tests for save/get/list/delete/export and a complete CLI flow. Implement a standard-library SQLite repository and `alaya` commands: `plant`, `reinforce`, `activate`, `list`, `show`, `export`, and `delete`.

## Task 5: Skill

Initialize `learn-from-experience` with the official skill initializer. Keep `SKILL.md` concise, route deterministic operations through a bundled script, document the candidate/evidence/human-review workflow, generate UI metadata, and run skill validation.

## Task 6: Public project surface

Write README, contribution guide, code of conduct, security policy, license, architecture notes, community-governance demo, evaluation script, and a truthful Codex for OSS application draft that explicitly says adoption evidence must be gathered before submission.

## Task 7: Verification

Run `python -m unittest discover -s tests -v`, the demo, the evaluation, package build checks available locally, schema parsing, CLI help, and skill validation. Fix failures, scan for placeholders and secrets, then prepare a single distributable archive and persistent deliverable.
