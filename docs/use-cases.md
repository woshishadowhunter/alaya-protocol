# Use Cases

Alaya Protocol is useful when an agent needs practical experience that remains inspectable, revisable, and bounded.

## Good Fits

- Long-running coding agents that should learn from repeated project outcomes.
- Agent retrospectives where a lesson needs independent evidence before activation.
- Local-first agent memory experiments that cannot send private traces to a hosted memory service.
- Multi-agent systems that need scoped trust evidence rather than global reputation.
- Research projects comparing deterministic memory policy with model-generated reflection.

## Poor Fits

- Storing raw transcripts without review.
- Personal profiling or covert preference inference.
- High-stakes legal, medical, financial, employment, or eligibility decisions.
- Replacing explicit project instructions with inferred behavior.
- Treating one LLM reflection as a durable truth.

## Example Maintainer Workflows

1. Store a candidate lesson after a failed project decision.
2. Add independent supporting or contradicting evidence from later outcomes.
3. Activate only experience that matches the current situation and applicability boundary.
4. Inspect why the engine returned a lesson.
5. Export or delete seeds when they are no longer appropriate.
