# Experience Model

- Fact: what happened or is true. Store elsewhere.
- Preference: what a user likes. Store only with appropriate consent.
- Rule: an explicit instruction. Put it in project guidance.
- Experience: a contextual lesson supported by an action and observed outcome.

Every candidate states situation, action, observable outcome, lesson, future guidance, applicability, and possible counterevidence. Repeated descriptions of one event are not independent evidence.

The default policy starts at 0.5 confidence. Independent support adds 0.15; contradiction subtracts 0.2. Two independent supports activate a seed. Confidence has a 180-day half-life. These are engineering defaults, not psychological claims.
